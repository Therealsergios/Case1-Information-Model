### Описание API

В рамках задачи по синхронизации значений атрибутов объектов системы 2 в атрибуты соответствующих объхектов системы 1 был реализован интеграционный сервис backend-to-backend. 

1. Сервис срабатывает по таймеру какждые сутки, собирает данные по объектам системы 1 с ключами к объектам системы 2
2. По ключам uuid сервис получает информацию о соответствущих объектах в системе 2, по которым есть изменения за прошедшие сутки и запрашивает перечень объектов и их атрибутов
3. Выполняет обновление атрибутов найденых объектов в системе 1

Полная OpenAPI 3.0 спецификация для REST API интеграционного сервиса для обогащения данными объектов PDM системы из ERP системы представлена [здесь](https://github.com/Therealsergios/Case1-Information-Model/blob/main/rest%20api/speci.yaml)
